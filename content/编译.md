## 1. 项目简介

WezTerm 是一个 GPU 加速的跨平台终端模拟器和多路复用器，使用 Rust 编写。工作区包含约 40 个 crate，采用 Cargo workspace 管理。

---

## 2. 编译环境

### 2.1 核心工具链

| 工具 | 版本 | 用途 |
|------|------|------|
| **rustc** | 1.95.0 (59807616 e 2026-04-14) | Rust 编译器 |
| **cargo** | 1.95.0 | Rust 包管理和构建系统 |
| **MSVC cl.exe** | 19.51.36243 (x 64) | C/C++ 编译器（原生依赖编译） |
| **CMake** | 4.2.3-msvc 3 | 部分原生依赖构建 |
| **Perl** | 5.42.2 (Strawberry Perl, MSWin 32-x 64-multi-thread) | OpenSSL 编译脚本所需 |
| **Git** | 2.54.0.windows.1 | 获取版本号、拉取子模块 |

### 2.2 Rust 目标与配置

- **Target**: `x86_64-pc-windows-msvc`
- **Profile**: `release` (`opt-level = 3`)
- **CRT**: 静态链接 (`+crt-static`)，在 `.cargo/config.toml` 中配置
- **OpenSSL**: 通过 `openssl-src` 从源码编译，静态链接

### 2.3 特殊工具路径

| 工具              | 路径                                            | 说明                            |
| --------------- | --------------------------------------------- | ----------------------------- |
| MSVC 环境配置       | `C:\Projects\cmds\VsDevCmd.ps1`               | 提供 cl.exe、INCLUDE、LIB 等环境变量   |
| Strawberry Perl | `C:\Users\<username>\Applications\perl-unzip` | 通过 PowerShell profile 注入 PATH |

> **注意**: MSVC 环境每次新建终端需先执行 `VsDevCmd.ps1`，否则 Rust 找不到链接器。
> Perl 的 PATH 通过 PowerShell 配置文件 `$PROFILE` 动态注入，不可使用 `-NoProfile` 参数。

## 3. 编译过程

### 3.1 编译命令

```powershell
# 1. 加载 MSVC 编译环境
& 'C:\Projects\cmds\VsDevCmd.ps1'

# 2. 编译 CLI 前端
cargo build -p wezterm --release

# 3. 编译 GUI 终端
cargo build -p wezterm-gui --release

# 4. 编译多路复用服务器
cargo build -p wezterm-mux-server --release
```

## 5. 排错记录

### 5.1 OpenSSL 编译失败

**现象**: `openssl-sys` 编译失败，错误信息 `Can't locate Locale/Maketext/Simple.pm in @INC`

**原因**: 系统中原本只有 PortableGit 附带的 Cygwin Perl（`/usr/bin/perl`），缺少 `Locale::Maketext::Simple` 等核心模块且无 CPAN，无法满足 OpenSSL 编译脚本的 Perl 依赖。

**解决**: 安装 Strawberry Perl（解压版 v 5.42.2 64-bit）到 `C:\Users\<username>\Applications\perl-unzip`，通过 PowerShell profile 将其 `perl\bin` 目录加入 `$env:PATH`。

**验证**: `perl -e 'use Locale::Maketext::Simple; print "OK\n"'` 返回 OK。

### 5.2 PowerShell 2>&1 导致错误判断

**现象**: cargo 输出显示 `Finished` 但脚本退出码非 0。

**原因**: PowerShell 的 `2>&1` 重定向将 cargo 的 stderr 警告流同时提升为 `ErrorRecord`，导致 `$?` 为 False。

**解决**: 直接检查产物 `target/release/wezterm.exe` 是否存在，或使用 `$LASTEXITCODE` 判断实际退出码。
