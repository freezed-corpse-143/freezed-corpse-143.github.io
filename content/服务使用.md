```javascript
function findDownloadButtonsStrict() {
    return Array.from(document.querySelectorAll(
        'button.el-button.el-button--text.el-button--small[type="button"] > span'
    ))
    .filter(span => span.textContent.trim() === '下载')
    .map(span => span.parentElement);
}
 
const downloadButtons = findDownloadButtonsStrict();

function clickFirstThreeButtons(downloadButtons) {
    if (downloadButtons.length === 0) {
        console.log('download button not found');
        return;
    }

    const totalGroups = Math.ceil(downloadButtons.length / 3);
    let currentGroup = 0;

    function clickNextGroup() {
        const start = currentGroup * 3;
        const end = Math.min(start + 3, downloadButtons.length);
        
        console.log(`clicking ${currentGroup + 1}/${totalGroups} group button (${start+1}-${end})`);
        
        function clickButton(index) {
            if (index >= end) {
                currentGroup++;
                if (currentGroup < totalGroups) {
                    setTimeout(clickNextGroup, 20000);
                } else {
                    console.log('all button clicked');
                }
                return;
            }
            
            try {
                console.log(`clicking  ${index - start + 1} -th button in group`);
                downloadButtons[index].click();
                setTimeout(() => clickButton(index + 1), 1000);
            } catch (error) {
                console.error(`clicking ${index + 1} -th button error:`, error);
                setTimeout(() => clickButton(index + 1), 1000);
            }
        }
        
        clickButton(start);
    }

    clickNextGroup();
}

clickFirstThreeButtons(downloadButtons);
```

## Blob 文件名的计算方式

根据文件是否通过 Git LFS 存储，分两种情况：

### 1. 小文件（Git 直接存储）→ SHA 1

文件名是 40 字符的 SHA 1 哈希，计算的是 Git blob 对象：

```
hash = SHA1("blob <字节数>\0<文件内容>")
```

**示例**：以 `config.json` 对应的 blob `9e26dfeeb6e641a33dae4961196235bdb965b21b` 为例

```python
import hashlib

content = b'{}'                          # 实际文件内容，2 字节
store   = b'blob 2\x00' + content        # Git blob 存储格式
hash    = hashlib.sha1(store).hexdigest()
# => 9e26dfeeb6e641a33dae4961196235bdb965b21b ✓ 匹配文件名
```

```
git hash-object <file_path>
```
### 2. 大文件（Git LFS 存储）→ SHA 256

文件名是 64 字符的 SHA 256 哈希，直接对原始文件内容计算：

```
hash = SHA256(file_content)
```

**示例**：以 `llm.pt` 对应的 blob 为例

```bash
$ sha256sum blobs/69f43bd545131c30e98947fb360ea8b4dc9916d8e83dded7757c7ea4f5a24970
69f43bd545131c30e98947fb360ea8b4dc9916d8e83dded7757c7ea4f5a24970  ← ✓ 一致
```

```hash
sha256sum <file_path>
```
### 总结

| 类型 | 哈希算法 | 计算内容 | 文件名长度 |
|------|----------|----------|------------|
| Git 直接存储（小文件） | SHA 1 | `blob <size>\0<content>` | 40 个十六进制字符 |
| Git LFS（大文件） | SHA 256 | 原始文件内容 | 64 个十六进制字符 |

> **说明**：这是完全遵循 Git / Git LFS 底层对象命名规则的。Hugging Face Hub 没有自己定义新的哈希方案，直接复用了 Git 的对象寻址机制。

## 模型文件结构

1. **`config.json`** - 模型架构配置（层数、注意力头数等）
2. **`model.safetensors`** 或 **`pytorch_model.bin`** - 模型权重（主要格式）
3. **分词器文件**：
   - `tokenizer.json` / `tokenizer_config.json` - 分词器配置
   - `vocab.json` / `vocab.txt` - 词汇表文件
   - `special_tokens_map.json` - 特殊标记映射
4. **`generation_config.json`** - 生成参数配置
5. **`README.md`** - 模型卡片/文档
6. **`model_index.json`** - 模型架构映射（部分模型）

## 模型格式差异

1. **SafeTensors (`.safetensors`) - 推荐使用**

- **安全性**：防止任意代码执行（不使用 pickle）
- **性能**：零拷贝加载，速度最快
- **文件大小**：与 PyTorch .bin 文件相似
- **加载方式**：`from_pretrained()` 自动优先使用此格式
- **安全性**：无恶意代码执行风险

2. **PyTorch (`.bin`) - 传统格式**

- **安全性**：**高风险** - 使用 pickle，可能执行任意代码
- **性能**：由于 pickle 反序列化，加载较慢
- **文件大小**：权重相同，序列化方式不同
- **加载方式**：如果 safetensors 不可用，则作为备选
- **安全性**：**避免从不可信来源下载**

3. **GGUF (`.gguf`) - 量化格式，适用于 CPU 推理**

- **目的**：针对 CPU 推理优化的量化格式
- **性能**：CPU 推理速度快，文件体积小
- **加载方式**：需要特殊处理（transformers 库不原生支持）
- **使用场景**：本地推理，如 llama.cpp、LocalAI 等
- **量化级别**：多种级别（Q4_K_M、Q8_0 等）

4. **GGML(.ggml) - 传统量化格式**

- **前身**：GGUF 的前身，现已逐渐被 GGUF 取代
- **特点**：为 CPU 推理优化的量化格式
- **使用场景**：llama.cpp 的早期版本
- **加载方式**：

```python
# 使用 llama.cpp
from llama_cpp import Llama
model = Llama(model_path="model.ggml")
```

5. **.pt/.pth**

- **标准格式**：PyTorch 的默认检查点格式

- **序列化**：使用 Python pickle

- **安全性**：**高风险**（可能执行任意代码）

- **保存方式**：

```python
torch.save(model.state_dict(), "model.pth")
```

- **加载方式**:

```python
model.load_state_dict(torch.load("model.pth"))
```

6. **.ckpt**

- **框架约定**：PyTorch Lightning 的标准命名

- **内容**：通常包含模型权重、优化器状态、训练进度

- **保存方式**：

```python
trainer.save_checkpoint("model.ckpt")
```

7. **.pb (SavedModel) - TensorFlow推荐格式**

- **完整模型**：包含计算图、变量、签名

- **可部署性**：可直接用于 TensorFlow Serving

- **保存方式**：

```python
tf.saved_model.save(model, "model_dir")
```

- **加载方式**：

```python
model = tf.saved_model.load("model_dir")
```

8. **.h5 / .hdf5 (Keras)**

- **Keras 格式**：HDF5 格式，用于 Keras 模型

- **内容**：模型架构、权重、优化器配置

- **保存方式**：

```python
model.save("model.h5")
```

- **加载方式**：

```python
model = tf.keras.models.load_model("model.h5")
```

9. **ONNX (.onnx) - 模型互操作性**

- **目的**：跨框架模型交换

- **支持框架**：PyTorch、TensorFlow、scikit-learn 等

- **转换示例**：

```python
# PyTorch → ONNX
torch.onnx.export(model, dummy_input, "model.onnx")

# TensorFlow → ONNX
import tf2onnx
tf2onnx.convert.from_keras(model, output_path="model.onnx")
```

- **推理**：

```python
import onnxruntime as ort
session = ort.InferenceSession("model.onnx")
outputs = session.run(None, {"input": input_data})
```

10. **Intel OpenVINO (.xml + .bin)**

- **优化格式**：针对 Intel 硬件优化

- **文件结构**：

- `.xml`：模型架构（IR 格式）

- `.bin`：模型权重

- **转换流程**：

```python
# 使用 OpenVINO Model Optimizer
mo --input_model model.onnx --output_dir openvino_model
```

- **推理**：

```python
from openvino.runtime import Core
core = Core()
model = core.read_model("model.xml")
compiled_model = core.compile_model(model, "CPU")
```

11. **Nvidia NeMo (.nemo)**

- **框架专用**：Nvidia NeMo 框架的模型格式

- **内容**：模型权重、配置、分词器、词汇表

- **保存方式**：

```python
model.save_to("model.nemo")
```

- **加载方式**：

```python
from nemo.collections import nlp as nemo_nlp
model = nemo_nlp.models.MTEncDecModel.restore_from("model.nemo")
```

## 系统指标测试

### /usr/bin/time

- 用法：`/usr/bin/time v program`
- 关键输出：
  - `Real`：**墙钟时间**，指从程序开始到结束所经过的实际时间，会受到系统上其他程序运行的影响。
  - `User`：**用户态CPU时间**，指程序自己的代码在用户空间运行所消耗的CPU时间。
  - `Sys`： **内核态CPU时间**，指程序执行内核调用（如文件读写、网络操作）所消耗的CPU时间。
  - `Maximum resident set size`：**程序生命周期中的最大物理内存占用**，单位通常是KB。

### perf

`perf`是Linux内核自带的强大性能分析工具，可以深入到CPU指令级别，帮助你找到程序的热点和瓶颈

- 安装：`apt install linux-tools-common linux-tools-$(uname -r)`
- 常用命令：
  - 整体统计：`perf stat porgram`，这会输出程序运行期间的各种硬件和软件事件计数，如CPU周期、指令数、分支预测失败、缓存未命中、上下文切换等
  - 性能剖析：`perf record -g ./program`，然后`perf report`查看。这会记录程序的调用图，并告诉你CPU时间主要花费在哪些函数上，以火焰图或树状结构展示。

### BenchExec

进行严谨的、可重复的基准测试，尤其是需要**精确限制和控制**被测程序的资源时，`BenchExec`是更专业的选择。它通过Linux的cgroups机制，可以准确测量包括子进程在内的所有资源占用

- 安装：`pip3 install benchexec`
- 特性：它能创建一个轻量级的“容器”来运行你的程序，确保测试结果不受系统其他活动的干扰，并能精确设置CPU时间、内存等上限。它通常通过编写一个简单的XML配置文件来定义测试任务。

### bandwidthTest

- 核心功能：通过使用**页锁定内存 (Pinned Memory)**，可以测量出PCIe的理论峰值带宽。这个值是你判断后续测试结果是否正常的“天花板”。

- 安装：下载 NVIDIA CUDA Samples

- 用法：在编译好CUDA Samples后，进入`bandwidthTest`目录，运行以下命令测试主机到GPU的单向带宽：

  ```bash
  ./bandwidthTest --memory=pinned --htod
  ```

### nvbandwidth

理论带宽测试无法完全模拟程序实际运行时的复杂情况。`nvbandwidth`工具可以模拟多种真实场景，帮你发现隐藏的瓶颈。

- 安装：下载 NVIDIA CUDA Samples
- **核心价值**：它不仅能测带宽，还能测**延迟**，并区分两种拷贝模式：
  - **Copy Engine (CE) 模式**：使用专用DMA引擎，不占用GPU计算核心，适合纯数据传输。
  - **Streaming Multiprocessor (SM) 模式**：使用GPU的计算核心来执行数据搬运，这更接近计算密集型和数据传输混合的真实场景。
- **关键用法**：编译后直接运行 `./nvbandwidth`，它会执行一系列全面的测试。你需要重点关注以下几项输出：
  - `host_to_device` 和 `device_to_host`：单项传输带宽。
  - `bidirectional` 传输带宽：这通常比单项传输带宽低，可以揭示总线在同时读写时的竞争情况。
  - `host_device_latency`：测量一次简单数据请求的往返时间，优秀的系统通常在500-800纳秒范围内。

### nvidia-smi

你的PCIe链路可能因为省电策略而在低速率下运行。在进行带宽测试时，需要同步监控链路状态，确保它在全速工作。

- 安装：下载 nvidia 的显卡驱动即可。

- **核心价值**：实时确认GPU当前的PCIe链路配置。

- **关键用法**：

  1. 打开一个新终端，运行监控命令：

     bash

     ```
     # 每秒查询一次PCIe信息
     watch -n 1 nvidia-smi -q -d PCI
     ```

     

  2. 在另一个终端运行你的带宽测试程序（如`bandwidthTest`）。

  3. 观察监控终端输出的关键字段，它们应该达到预期值：

     - **Current Link Width**：应为 `16x`。
     - **Current Link Speed**：应为 `16.0 GT/s` (PCIe 4.0) 或 `32.0 GT/s` (PCIe 5.0)。
     - **PCIe Rx/Tx Utilization**：显示了当前链路的实时使用率。

## 自动编号合并

```powershell
<#
.SYNOPSIS
Merge split Markdown files by base name (clean version)

.DESCRIPTION
Auto detect files ending with "_数字-数字.md" in current directory,
group by base name (prefix before the first "_-数字"),
sort by start page within each group, merge into separate MD files.
No extra comments or separators added.

.EXAMPLE
.\Merge-MarkdownParts.ps1
Merges:
  An_Introduction_to_Random_Matrices_1-200.md + ... -> An_Introduction_to_Random_Matrices.md
  Concentration_inequalities_1-200.md + ... -> Concentration_inequalities.md
#>

$scriptDir = Split-Path -Parent $MyInvocation.MyCommand.Path
Set-Location $scriptDir

# Get all files matching the pattern
$files = Get-ChildItem -Filter "*.md" | Where-Object { $_.Name -match '^(.+)_(\d+)-(\d+)\.md$' }

if ($files.Count -eq 0) {
    Write-Host "No split markdown files found (pattern: *_数字-数字.md)"
    exit
}

# Group by base name (the part before the last "_数字-数字")
$grouped = $files | ForEach-Object {
    if ($_.Name -match '^(.+)_(\d+)-(\d+)\.md$') {
        $baseName = $matches[1]
        $startPage = [int]$matches[2]
        $endPage = [int]$matches[3]
    }
    [PSCustomObject]@{
        File = $_
        BaseName = $baseName
        StartPage = $startPage
        EndPage = $endPage
    }
} | Group-Object BaseName

# Process each group
foreach ($group in $grouped) {
    $baseName = $group.Name
    $outputFile = "$baseName.md"
    
    # Sort files within group by start page
    $sortedFiles = $group.Group | Sort-Object StartPage | Select-Object -ExpandProperty File
    
    Write-Host "Merging $($sortedFiles.Count) files into $outputFile"
    
    # Remove existing output file if present
    if (Test-Path $outputFile) {
        Remove-Item $outputFile -Force
        Write-Host "  Removed existing $outputFile"
    }
    
    # Append content of each file
    foreach ($file in $sortedFiles) {
        Write-Host "  Adding: $($file.Name) (pages $($file.StartPage)-$($file.EndPage))"
        $content = Get-Content -Path $file.FullName -Raw -Encoding UTF8
        $content | Out-File -FilePath $outputFile -Append -Encoding UTF8 -NoNewline
    }
    
    Write-Host "Created: $outputFile"
    Write-Host ""
}

Write-Host "Merge completed!"
```
## 自动对比翻译目录与原目录

```powershell
param(
    [Parameter(Mandatory=$true)]
    [string]$DirectoryA,
    
    [Parameter(Mandatory=$true)]
    [string]$DirectoryB
)

# 验证目录是否存在
if (-not (Test-Path $DirectoryA -PathType Container)) {
    Write-Error "Directory A '$DirectoryA' does not exist or  is not directory"
    exit 1
}

if (-not (Test-Path $DirectoryB -PathType Container)) {
    Write-Error "Directory B '$DirectoryB' does not exist or  is not directory"
    exit 1
}

# 获取A目录下所有不以_zh.md结尾的文件
$files = Get-ChildItem -Path $DirectoryA -File | Where-Object { $_.Name -notlike "*_zh.md" }

foreach ($file in $files) {
    # 构建对应的_zh文件名
    $baseName = $file.BaseName
    $zhFileName = "${baseName}_zh.md"
    $zhFilePath = Join-Path $DirectoryA $zhFileName
    
    # 检查_zh文件是否存在
    if (Test-Path $zhFilePath -PathType Leaf) {
        # 检查B目录中是否存在同名文件
        $targetFile = Join-Path $DirectoryB $file.Name
        if (Test-Path $targetFile -PathType Leaf) {
            # 复制_zh文件到B目录
            $targetZhFile = Join-Path $DirectoryB $zhFileName
            Copy-Item -Path $zhFilePath -Destination $targetZhFile -Force
            Write-Host "Copied: $zhFileName 到 $DirectoryB"
        }
    }
}

Write-Host "Well done"

```

## API 概览

MinerU 提供两套 API：

| API                                     | 认证                | 限制           | 适用场景         |
| --------------------------------------- | ------------------- | -------------- | ---------------- |
| **Agent API** (`/api/v1/agent/parse/*`) | 无需认证（IP 限频） | ≤10MB, ≤20页   | 小文件快速转换   |
| **Precision API** (`/api/v4/extract/*`) | Bearer Token        | ≤200MB, ≤200页 | 大文件、批量转换 |

### Precision API（推荐，功能完整）

需要 Token 认证，支持 URL 提交和本地文件上传两种模式。

**URL 模式** — 直接提交 PDF 链接：

```
POST https://mineru.net/api/v4/extract/task
Headers:
  Authorization: Bearer <token>
  Content-Type: application/json
Body:
  {
    "url": "https://arxiv.org/pdf/2307.08621",
    "language": "en",
    "model_version": "pipeline"     // pipeline | vlm | MinerU-HTML
  }
```

**轮询结果**：

```
GET https://mineru.net/api/v4/extract/task/{task_id}
Headers:
  Authorization: Bearer <token>
```

状态 (`state`) 流转: `pending` → `running` → `done` / `failed`

完成后返回 `full_zip_url`，解压得到：

- `full.md` — 完整 Markdown 文档
- `images/` — 提取的图片
- `layout.json` — 布局信息
- `*_content_list.json` — 内容列表
- `*_origin.pdf` — 原始 PDF

### Agent API

```
POST https://mineru.net/api/v1/agent/parse/file
Headers: Content-Type: application/json
Body: {"file_name": "paper.pdf", "language": "en"}

# 返回 task_id 和 file_url，PUT 上传文件后自动开始解析
# 轮询 GET /api/v1/agent/parse/{task_id}，完成后返回 markdown_url
```

### 关键参数

| 参数              | 说明             | 可选值                                                       |
| ----------------- | ---------------- | ------------------------------------------------------------ |
| `language`        | 文档语言         | `en`(英文), `ch`(中英混合,默认), `japan`, `korean`, `latin` 等 |
| `model_version`   | 解析模型         | `pipeline`(默认), `vlm`(视觉语言模型), `MinerU-HTML`         |
| `enable_formula`  | 启用公式提取     | `true`(默认), `false`                                        |
| `enable_table`    | 启用表格提取     | `true`(默认), `false`                                        |
| `is_ocr`          | 启用 OCR         | `true`, `false`(默认)                                        |
| `extra_formats`   | 额外输出格式     | `["docx", "html", "latex"]`                                  |
| `no_cache`        | 跳过缓存         | `true`, `false`(默认)                                        |
| `cache_tolerance` | 缓存容忍时间(秒) | 默认 `900`                                                   |

### 常见错误码

| 错误码   | 含义                                  |
| -------- | ------------------------------------- |
| `A0202`  | Token 格式错误（检查 `Bearer ` 前缀） |
| `A0211`  | Token 已过期                          |
| `-60005` | 文件超过 200MB                        |
| `-60006` | 超过 200 页                           |
| `-60018` | 每日用量用尽（1000 优先页）           |
| `429`    | Agent API IP 限频                     |
