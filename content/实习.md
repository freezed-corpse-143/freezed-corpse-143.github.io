# 上传文件

1. 创建 `gitlab token`：进入 git.nevint.com/-/profile → 左侧边栏 Access Tokens → Personal Access Tokens → Add new token。保存为环境变量 `$GITLAB_TOKEN`
2. 获取项目 ID 为 环境变量$PROJECT_ID

```bash
curl --slient --header "PRIVATE-TOKEN: $GITLAB_TOKEN""https://gitlab.com/api/v4/projects/$OWNER%2F$PROJECT_NAME" | python3 -c "import sys,json; print(json.load(sys.stdin)['id'])"
```
3. 上传文件：

```bash
curl --location --header "PRIVATE-TOKEN: $GITLAB_TOKEN" --upload-file "/path/to/file.zip" "https://gitlab.com/api/v4/projects/$PROJECT_ID/packages/generic/$PACKAGE_NAME/$PACKAGE_VERSION/file.zip"
```
4. 下载地址：
`https://gitlab.com/api/v4/projects/$PROJECT_ID/packages/generic/$PACKAGE_NAME/$PACKAGE_VERSION/file.zip`

# 下载仓库文件

> GITLAB_TOKEN 需要具有 `read_repository` 权限

在执行路径下配置 `.env`，并写入环境变量 `$GITLAB_TOKEN`

```bash
. .env && curl --header "PRIVATE-TOKEN: $GITLAB_TOKEN" -fsSL "https://gitlab.com/api/v4/projects/<PROJECT_ID>/repository/files/install.sh/raw?ref=main" | bash
```

# 克隆仓库

> GITLAB_TOKEN 需要具有 `read_repository` 权限

在执行路径下配置 `.env`，并写入环境变量 `$GITLAB_TOKEN`

```bash
git clone https://oauth2:${GITLAB_TOKEN}@git.nevint.com/<owner>/<repo>.git
```

# 如何通过 SKILL 快速部署

1. 将 GITLAB_TOKEN，保存在 Agent 环境中的文件中，并暴露位置给 Agent。
2. 在 SKILL 中告诉 Agent：
	1. 去本地拿 GITLAB_TOKEN
	2. 克隆项目。
	3. 执行 Install.sh。
# 切换默认分支
