# RSS订阅

一句话概括：**RSS是一种让你“主动订阅，自主阅读”的信息分发技术。**

-   **本质**：它是一个遵循特定格式（XML）的文件，网站用它来发布最新内容（标题、摘要、链接）。
-   **核心价值**：**把信息选择权还给你**。
    -   **告别算法**：你看什么，由你订阅了谁决定，而不是平台算法。
    -   **告别广告**：RSS阅读器通常只展示纯净的内容。
    -   **统一看板**：将你关注的新闻、博客、视频更新等，聚合在一个地方阅读。

你可以把它理解为“**信息世界的订报服务**”——你决定订哪几份报纸（订阅源），报亭每天准时把报纸合在一起送到你家（阅读器聚合）。

## 怎么使用

核心就三步：**找源 → 加源 → 读源**。

**第一步：获取RSS链接（找“报纸的投递地址”）**

-   **方法A（最省事）**：寻找网站上的橙色RSS图标（），通常位于页脚或侧边栏。
-   **方法B（最强大）**：使用开源工具 **RSSHub**。对于不支持RSS的网站（如B站、微博），按规则拼接一个链接即可生成RSS源。
    -   例如，B站UP主视频：`https：//rsshub.app/bilibili/user/video/22675757` （数字换成你想关注的UP主ID）
-   **方法C（最便捷）**：安装浏览器扩展 **RSSHub Radar**，它能自动发现当前页面的RSS源。

**第二步：选择阅读器（“订报箱”）**

找一个开源的阅读器，把你的RSS链接添加进去。

-   **电脑全平台（Win/Mac/Linux）**：推荐 **RSS Guard**，功能强大。
-   **苹果用户（Mac/iOS）**：推荐 **NetNewsWire**，原生流畅，体验极佳。
-   **安卓用户**：推荐 **Capy Reader**，轻量简洁。

**第三步：开始阅读**

-   将你找到的RSS链接，粘贴到阅读器的“添加订阅”或“New Feed”中。
-   阅读器会自动抓取所有你订阅的源，并按时间顺序展示最新内容。

**一个典型的RSS工作流是这样的：**

> 你发现了一个不错的科技博客 → 找到它的RSS链接（如 `https：//blog.example.com/feed`）→ 打开你的RSS阅读器（如 RSS Guard）→ 添加这个链接 → 从此以后，这个博客每次更新，都会自动出现在你的阅读器里，和其他所有订阅源混排，供你集中阅读。

总结就是：**RSS让你不再被动等待算法投喂，而是主动构建并享受一个安静、高效、属于自己的信息世界。**

## 为网站生成RSS链接

如果一个网站本身不提供RSS订阅源，你可以使用 **RSSHub** 来“制造”一个RSS链接。

RSSHub 是什么？

RSSHub 是一个开源、免费的RSS生成器，能够为数百个网站的上千项内容生成RSS订阅源。简单说，它像一个“翻译官”，把B站、微博、知乎、微信公众号等原本不支持RSS的网站内容，转换成标准的RSS格式。

如何使用 RSSHub？

**方法一：使用公共实例（推荐新手）**

RSSHub 提供了免费公共实例，你无需安装任何东西，直接按规则拼接链接即可使用：

| 公共实例地址                    | 说明               |
| :------------------------------ | :----------------- |
| `https://rsshub.app`            | 官方主实例，最常用 |
| `https://rsshub.rssforever.com` | 备用实例           |
| `https://hub.slarker.me`        | 备用实例           |

**使用步骤**：

1. 找到你想订阅的内容对应的“路由规则”
2. 将路由规则拼接到公共实例地址后面
3. 把完整链接添加到你的RSS阅读器

**方法二：本地自部署（适合技术用户）**

如果你希望数据完全自主可控，可以用 Docker 在本地部署自己的 RSSHub 实例：

```bash
# 拉取镜像
sudo docker pull diygod/rsshub

# 运行容器（访问端口1200）
sudo docker run -d --name rsshub -p 1200:1200 diygod/rsshub
```

部署完成后，在浏览器访问 `http://localhost:1200` 即可看到RSSHub界面，然后用 `http://localhost:1200/路由规则` 来订阅内容。

- **B站UID**：进入UP主主页，浏览器地址栏 `https://space.bilibili.com/22675757` 中的数字就是UID
- **微博UID**：打开博主主页，在浏览器开发者工具控制台输入 `$CONFIG.oid` 即可获取
- **知乎ID**：用户主页URL中 `https://www.zhihu.com/people/zhang-jia-wei` 后面的部分是ID

## 跟踪网站变化

方法一：使用 RSSHub-Radar 浏览器扩展（自动发现）

**RSSHub-Radar** 是一款浏览器扩展，能够像雷达一样自动扫描当前网页，发现所有可用的RSS订阅源（包括原生RSS和RSSHub支持的路由）。

**核心功能**：

- **自动识别**：当你访问一个网站时，扩展自动检测该网站是否支持RSS订阅
- **一键订阅**：点击扩展图标，所有可订阅源清晰展示，选择后直接添加到你的RSS阅读器
- **智能发现**：即使网站没有原生RSS，也能检测是否被RSSHub支持

**使用步骤**：

1. 在 Chrome/Firefox 应用商店搜索安装 **RSSHub-Radar**
2. 正常浏览网页，扩展图标亮起时点击
3. 在弹出的订阅源列表中选择感兴趣的内容
4. 点击订阅，即可添加到RSS阅读器

**适用场景**：

- 浏览到优质博客/网站时，快速订阅
- 不确定网站是否支持RSS，让扩展自动帮你发现

方法二：利用 RSS Guard 自动抓取

RSS Guard 阅读器本身就支持自动跟踪网站变化：

1. 进入菜单 **Tools → Settings → Feeds & articles**
2. 设置 **Auto-fetch interval**（自动获取间隔），比如设为30分钟
3. RSS Guard 会按设定时间自动检查所有订阅源是否有新内容

方法三：使用 Changedetection.io（专门跟踪网页变化）

如果你需要跟踪的网站没有RSS、也不被RSSHub支持，可以使用 **Changedetection.io** 这个开源工具。它会定期抓取指定网页，检测内容是否发生变化，发生变化时通过邮件、RSS等方式通知你。

**适用场景**：

- 跟踪商品价格变化
- 监控招聘页面更新
- 关注政府公告/政策页面
